# 表情识别问题诊断快速指南

## 问题描述
表情控制状态显示一直为"平静"，无法识别不同的面部表情。

## 快速诊断（3 步）

### 步骤 1：准备诊断环境

在终端执行：

```powershell
cd c:\Users\acer\Desktop\BS\Coding
.\.venv312\Scripts\Activate.ps1
```

### 步骤 2：运行诊断脚本

```powershell
python scripts/diagnose_expression.py
```

脚本会打开摄像头视图窗口，并每 10 帧输出一次诊断数据。

### 步骤 3：测试表情变化

在摄像头前执行以下动作，观察输出数据的变化：

| 表情 | 动作 | 应该识别为 | 关键参数 |
|------|------|----------|---------|
| 咧嘴大笑 | 张大嘴巴，嘴角向两侧张开 | 咧嘴大笑 | MAR ↑ + 嘴宽比 ↑ |
| 皱眉 | 眉毛下压，嘴角下垂 | 愤怒 | 嘴角抬升 ↓ |
| 惊讶 | 张大嘴巴，但嘴角不水平张开 | 惊讶 | MAR ↑ + 嘴宽比 ↓ |
| 微笑 | 轻微张嘴，嘴角上扬 | 抿嘴微笑 | 嘴角抬升 ↑ |

## 诊断数据解释

### 如果看到以下情况，说明...

**情况 1：所有表情参数都很低**
```
MAR: 0.08 | 嘴宽比: 0.30 | 嘴角抬升: 0.002
→ 原因：摄像头距离过远或光线不足
→ 解决：靠近摄像头 20-40cm，改善光线
```

**情况 2：MAR 很高，但嘴宽比很低**
```
MAR: 0.22 | 嘴宽比: 0.35 | 嘴角抬升: 0.001
→ 原因：嘴巴张开（纵向），但没有水平张开
→ 解决：做出更"夸张"的表情，横向张开嘴角
```

**情况 3：参数都能正确变化，但仍显示"平静"**
```
咧嘴大笑时: MAR: 0.25, 嘴宽比: 0.52
→ 应该识别为"咧嘴大笑"，但显示"平静"
→ 原因：识别阈值设置过高，需要调整
→ 解决：见下一节
```

## 调整识别阈值

如果诊断工具显示参数正常变化，但仍无法识别表情，需要调整阈值。

### 方案 A：降低全局灵敏度（推荐）

编辑 `Face/PythonProject/config.py`：

```python
# 找到这一行
'MOUTH_THRESHOLD': 0.13,

# 改为更低的值（提高灵敏度）
'MOUTH_THRESHOLD': 0.10,  # 试试这个
```

### 方案 B：微调具体表情的阈值（精细调整）

编辑 `Face/PythonProject/face_analysis.py` 中的 `detect_face_expression` 函数。

例如，如果咧嘴大笑无法识别，修改这一行：

```python
# 原：
if mar >= max(mouth_threshold * 1.55, 0.18) and mouth_width_ratio >= 0.48:

# 改为（更宽松）：
if mar >= max(mouth_threshold * 1.30, 0.15) and mouth_width_ratio >= 0.42:
```

**调整建议**：
- 如果某个表情无法识别，将 `口 * 系数` 中的系数降低 10-15%
- 将固定阈值（如 0.18、0.48）也略微降低

### 方案 C：启用实时诊断日志

编辑 `Face/PythonProject/face_analysis.py`，找到这一行（约第 92 行）：

```python
# 诊断输出（临时用于调试）
# print(f"[表情诊断] MAR={mar:.4f} 嘴宽比={mouth_width_ratio:.4f} 嘴角抬升={corner_lift:.6f}")
```

移除注释符号：

```python
# 诊断输出（临时用于调试）
print(f"[表情诊断] MAR={mar:.4f} 嘴宽比={mouth_width_ratio:.4f} 嘴角抬升={corner_lift:.6f}")
```

然后重启 face 模块，每帧都会在控制台输出参数值。

## 修改后的验证步骤

1. 重启 face 模块（停止后重新启动）
2. 再次运行诊断脚本（或打开面部页面）
3. 做出各种表情，观察是否能被正确识别
