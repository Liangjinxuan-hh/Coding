# 3.2 面部识别方法

本系统的面部识别模块采用基于人脸关键点几何特征的状态识别方法。与身份识别任务不同，本研究关注“交互状态识别”，即根据人脸关键点的相对位置关系识别用户当前的眼部、口部与头部姿态状态，并映射为控制指令。该方法具有实时性高、可解释性强、部署成本低等特点，适用于本系统的实时交互场景。

## 3.2.1 方法原理

系统首先通过 MediaPipe Face Mesh/FaceLandmarker 获取单人脸关键点坐标，然后计算眼睛开合比（EAR）、嘴部开合比（MAR）及头部方向偏移量等几何特征。最后结合动态阈值与规则判定机制，输出交互状态（如睁眼/闭眼、张嘴/闭嘴、左转/右转、抬头/低头），并进一步转换为标准动作命令。

该流程可概括为：

1. 关键点检测：实时获取人脸关键点坐标。
2. 几何特征计算：计算 EAR、MAR、水平与竖直姿态特征。
3. 阈值判定：根据动态或固定阈值进行状态分类。
4. 状态映射：将分类结果映射为控制命令。

## 3.2.2 关键公式

1. 眼睛开合比（EAR）定义为：

$$
EAR=\frac{\lVert P_1-P_5\rVert+\lVert P_2-P_4\rVert}{2\lVert P_0-P_3\rVert}
\tag{3-1}
$$

式中，$P_0\sim P_5$ 为单眼六个关键点，$\lVert\cdot\rVert$ 表示欧氏距离。

2. 动态闭眼阈值定义为：

$$
T_{eye}=EAR_{base}\times S_{eye}
\tag{3-2}
$$

其中，$EAR_{base}$ 为个体基准睁眼值（由校准阶段统计得到），$S_{eye}$ 为眼部灵敏度系数。

3. 嘴部开合比（MAR）定义为：

$$
MAR=\frac{|y_{upper}-y_{lower}|}{|x_{left}-x_{right}|}
\tag{3-3}
$$

其中，$y_{upper}$、$y_{lower}$ 分别为上、下唇中心纵坐标，$x_{left}$、$x_{right}$ 分别为左右嘴角横坐标。

4. 头部水平转向特征定义为：

$$
\Delta x=x_{nose}-\frac{x_{eyeL}+x_{eyeR}}{2}
\tag{3-4}
$$

当 $\Delta x$ 大于正阈值判定为右转，小于负阈值判定为左转，介于阈值之间判定为前向。

5. 头部竖直姿态特征定义为：

$$
d_{nc}=y_{chin}-y_{nose}
\tag{3-5}
$$

当 $d_{nc}$ 超过上阈值时判定为低头；当 $d_{nc}$ 小于下阈值时判定为抬头；否则判定为平视。

## 3.2.3 动态阈值与鲁棒性设计

为降低个体差异和环境变化对识别精度的影响，系统引入动态阈值校准机制：

1. 在初始化阶段采集连续若干帧 EAR 样本。
2. 取最近校准窗口内 EAR 的高分位统计值作为基准睁眼值。
3. 使用式（3-2）生成个体化闭眼阈值。

此外，系统对状态切换设置了冷却时间窗口，避免由于短时抖动造成命令频繁触发，从而提高交互稳定性。

## 3.2.4 人脸关键点定义

系统采用 MediaPipe Face Mesh 的固定索引定义，关键点如下。

| 类别 | 关键点索引 | 语义说明 |
|---|---|---|
| 左眼 EAR 六点 | 362, 385, 387, 263, 373, 380 | 左眼开合几何计算 |
| 右眼 EAR 六点 | 33, 159, 158, 133, 145, 144 | 右眼开合几何计算 |
| 上下唇中心 | 13, 14 | 嘴部垂直开合距离 |
| 左右嘴角 | 61, 291 | 嘴部水平基准宽度 |
| 鼻尖 | 4 | 头部方向与姿态参照点 |
| 左右眼角 | 33, 263 | 水平朝向中心参照 |
| 下颌中心 | 152 | 竖直姿态判定参照 |
| 额头中心 | 10 | 辅助竖直姿态分析 |

## 3.2.5 本系统中的状态映射

面部识别结果在系统中映射为交互命令，典型映射关系如下：

1. 眼部状态：睁眼/闭眼 -> OPEN_EYES / CLOSE_EYES。
2. 嘴部状态：张嘴/闭嘴 -> OPEN_MOUTH / CLOSE_MOUTH。
3. 水平姿态：左转/右转/前向 -> LOOK_LEFT / LOOK_RIGHT / DEFAULT。
4. 竖直姿态：抬头/低头/平视 -> LOOK_UP / LOOK_DOWN / LOOK_STRAIGHT。
5. 左右眼组合：A/B/C/D 模式 -> SELECT_RING_A/B/C/D。

## 3.2.6 小结

本节提出的关键点几何特征识别方法，通过 EAR、MAR 与头部姿态特征实现了轻量级、可解释的实时面部交互识别。结合动态阈值校准与状态防抖策略，系统在保证响应速度的同时具备较好的稳定性，为后续动作控制与多模态融合提供了可靠输入。
